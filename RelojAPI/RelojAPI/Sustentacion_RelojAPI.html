Diapositiva 2 — Integrantes

Diapositiva 3 — Idea del Proyecto
"La idea fue construir un sistema completo con dos partes conectadas. El backend expone una API REST con tres entidades principales: Marcas, Relojes y Usuarios. Implementamos CRUD completo, autenticación con hash de contraseñas, un sistema de rol administrador, subida de imágenes, y un frontend React que consume la API en tiempo real sin ningún proceso de build — todo desde un solo archivo HTML."
Diapositiva 4 — Arquitectura Backend
"La estructura del backend sigue una arquitectura por capas. Los Controllers reciben las peticiones HTTP y devuelven JSON. Los Services contienen la lógica de negocio y están desacoplados mediante interfaces. Los Models son las clases C# que EF Core convierte en tablas. El AppDbContext en la carpeta Data es el puente entre el código y SQL Server. Y en wwwroot/images se almacenan las imágenes subidas."
Diapositiva 5 — Arquitectura Frontend
"El frontend vive en un solo archivo index.html. Usamos React 18 y Babel standalone cargados desde CDN — el navegador transforma el JSX directamente, sin necesidad de Node.js ni ningún proceso de compilación. Lo servimos con dotnet-serve en el puerto 5500, que hace peticiones a la API en el puerto 5157."
Diapositiva 6 — Modelo de Datos
"Tenemos tres tablas. Marcas almacena el fabricante. Relojes tiene relación N a 1 con Marcas a través de una llave foránea MarcaId, e incluye el campo ImagenUrl para guardar la ruta de la foto. Usuarios tiene un índice único en Email, almacena el hash de la contraseña, y tiene los campos Activo y EsAdmin como bits para controlar el acceso."
Diapositiva 7 — EF Core Migrations
"Usamos el enfoque Code-First de Entity Framework Core. Esto significa que primero escribimos las clases en C# y luego EF Core genera el SQL para crear las tablas. Con dotnet ef migrations add generamos el código de la migración, y con dotnet ef database update lo aplicamos a SQL Server. EF Core mantiene una tabla llamada __EFMigrationsHistory para saber qué migraciones ya se aplicaron."
Diapositiva 8 — Librerías NuGet
"Instalamos cuatro paquetes. EntityFrameworkCore.SqlServer es el ORM que mapea nuestras clases a tablas. EntityFrameworkCore.Tools habilita los comandos dotnet ef en la terminal. Swashbuckle.AspNetCore genera la interfaz Swagger UI automáticamente a partir de nuestros controllers. Y Microsoft.AspNetCore.OpenApi proporciona soporte nativo de OpenAPI en .NET 9."
Diapositiva 9 — Program.cs Parte 1: Servicios
"En Program.cs registramos todos los servicios del sistema. AddCors define la política que permite al frontend en el puerto 5500 comunicarse con la API. AddJsonOptions con IgnoreCycles evita un error de serialización circular que ocurre porque Reloj tiene una referencia a Marca y Marca tiene una lista de Relojes. AddDbContext conecta EF Core con la cadena de conexión de SQL Server. Y AddScoped registra nuestro servicio personalizado en el contenedor de inyección de dependencias."
Diapositiva 10 — Program.cs Parte 2: Pipeline
"El pipeline define el orden en que el sistema procesa cada petición HTTP. Primero exponemos Swagger en la raíz. Luego aplicamos CORS. UseStaticFiles es el middleware que sirve las imágenes subidas desde wwwroot/images directamente como archivos estáticos. MapControllers enruta cada petición HTTP al controlador correspondiente según la URL."
Diapositiva 11 — Program.cs Parte 3: Diagrama de flujo
"Visualmente, cada request pasa por una cadena: entra, pasa por CORS, luego por los archivos estáticos, luego por autorización, llega al Controller, baja al Service, el Service usa el DbContext para hablar con SQL Server, y el resultado sube de regreso como JSON al navegador. El orden del pipeline es crítico — si CORS no está antes de los controllers, el navegador rechaza las respuestas."
Diapositiva 12 — Inyección de Dependencias: Concepto
"La Inyección de Dependencias es uno de los pilares de ASP.NET Core. En lugar de que cada clase cree sus propias dependencias con new, registramos todo en Program.cs y el framework se encarga de inyectarlas automáticamente donde se necesiten. En nuestro caso, IRelojService, AppDbContext, ILogger e IConfiguration están disponibles en cualquier controlador sin instanciarlos manualmente."
Diapositiva 13 — Inyección de Dependencias: Código
"Así se ve en código. El constructor de RelojController declara tres parámetros: IRelojService para la lógica de negocio, ILogger para registrar eventos en consola, e IConfiguration para leer valores del appsettings.json. ASP.NET Core detecta esos parámetros, los resuelve desde el contenedor, y los pasa al constructor automáticamente en cada petición. Con esto cedo la palabra a Axel."

"Para los usuarios implementamos registro y login. La seguridad se basa en SHA256: nunca guardamos la contraseña en texto plano — guardamos su hash. Diapositiva 17 — Conexión a SQL Server
"La cadena de conexión en appsettings.json apunta a localhost\SQLEXPRESS, base de datos RelojDB, con autenticación de Windows. El AppDbContext hereda de DbContext y tiene tres DbSet que mapean a las tablas. Diapositiva 22 — CORS
"CORS fue necesario porque el navegador, por seguridad, bloquea peticiones entre diferentes orígenes. Nuestro frontend corre en el puerto 5500 y la API en 5157 — son orígenes distintos. Sin la política CORS configurada en Program.cs, el navegador rechazaría todas las respuestas de la API aunque la petición llegara al servidor. En desarrollo usamos AllowAnyOrigin, pero en producción se limitaría al dominio real."

CÓMO CREAMOS EL PROYECTO — Resumen técnico

1 · Crear el proyecto API en Visual Studio 2022

Nuevo proyecto → ASP.NET Core Web API → .NET 9 → Sin autenticación. Esto genera la estructura base con Program.cs y la carpeta Controllers.

2 · Instalar paquetes NuGet

dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Tools
dotnet add package Swashbuckle.AspNetCore
dotnet add package Microsoft.AspNetCore.OpenApi

3 · Crear los Modelos (C# → SQL Server)

Escribimos clases C# en la carpeta Models/. EF Core las convierte en tablas automáticamente.

// Models/Reloj.cs → tabla "Relojes"
public class Reloj {
    public int Id { get; set; }
    public string Nombre { get; set; } = string.Empty;
    public decimal Precio { get; set; }
    public int Stock { get; set; }
    public string? ImagenUrl { get; set; }
    public int MarcaId { get; set; }        // llave foránea
    public Marca? Marca { get; set; }       // navegación 1:N
}

4 · Crear el AppDbContext (el puente C# ↔ SQL Server)

public class AppDbContext : DbContext {
    public DbSet<Marca> Marcas { get; set; }
    public DbSet<Reloj> Relojes { get; set; }
    public DbSet<Usuario> Usuarios { get; set; }
}

5 · Configurar la cadena de conexión en appsettings.json

"ConnectionStrings": {
  "DefaultConnection": "Server=localhost\\SQLEXPRESS;
    Database=RelojDB;Trusted_Connection=True;
    TrustServerCertificate=True;"
}

6 · Registrar todo en Program.cs

builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(
        builder.Configuration.GetConnectionString("DefaultConnection")
    ));

7 · Correr las Migraciones (crear la BD física)

dotnet ef migrations add InitialCreate   // genera el código SQL
dotnet ef migrations add AddUsuarios     // agrega tabla Usuarios
dotnet ef database update                // aplica todo a SQL Server

Estos comandos crearon las tablas Marcas, Relojes y Usuarios en SQL Server sin escribir SQL manualmente.

8 · Los Controllers usan el Context vía Inyección de Dependencias

public class RelojController : ControllerBase {
    private readonly IRelojService _service;

    public RelojController(IRelojService service) {
        _service = service; // ASP.NET lo inyecta automáticamente
    }

    [HttpGet]
    public async Task<IActionResult> GetAll() {
        var relojes = await _service.GetAllAsync();
        return Ok(relojes); // devuelve JSON
    }
}

9 · Flujo completo resumido

React (fetch)
  → Controller (recibe la petición HTTP)
    → Service (lógica de negocio)
      → AppDbContext (traduce a SQL con EF Core)
        → SQL Server (guarda / lee datos)
      ← devuelve objetos C#
    ← devuelve objetos C#
  ← devuelve JSON al navegador → React renderiza
